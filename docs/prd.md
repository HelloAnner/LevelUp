## 0. 共创洞察摘要

### 核心洞察

1. 用户有目标，缺的不是工具——是「有人陪他把目标执行下去」的关系感
2. AI 陪伴产品有温度无系统；目标工具有系统无温度——中间是真实空白
3. 「关系 × 系统」是双护城河：关系阻止流失，系统阻止复制
4. Web 端优先：目标管理需要大屏输入和可视化
5. 对话是第一界面，Sidebar 是系统容器——用户不主动管理系统，系统由 AI 在对话里浮现
6. 目标层级树（意义感） + 实施意图（执行力）是科学且可产品化的方法论
7. 卡片嵌入对话是 MVP 阶段最低成本、最高价值的系统可见性方案
8. 用户为「可见的成长结果」付费，不为「功能」付费
9. AI 人格一致性是留存核心变量，也是最大技术风险
10. 首批用户来自「正在执行一个重要目标」的人群，不是泛成长人群
11. 「当 X 时我就 Y」结构天然适合对话式生成
12. 路线图可视化是产品「哇时刻」，作为 Phase 2 的重磅转化杠杆

### 被排除的方向

- 移动端优先 · 纯习惯追踪器 · 纯 AI 聊天机器人 · ToB 企业版 · 社区 / 好友功能

## 1. 产品概述

### 1.1 一句话定义

为 **有重要目标在执行中的 25–35 岁知识工作者**，提供 **像朋友一样陪伴、像系统一样运作的 AI 成长伙伴**，通过 **「目标层级树 + 实施意图」双框架驱动的对话式目标管理**，解决 **「有目标、无系统、无人陪」的执行力黑洞**。

### 1.2 产品定位

- **目标市场**：正在执行一个重要人生目标的 25–35 岁中英文知识工作者（创业、转行、技能提升、写作出版等）
- **价值主张**：你的 AI 成长伙伴——懂你的目标，陪你走完全程
- **护城河**：
  - 短期：AI 人格一致性 + 目标数据积累
  - 中期：目标层级树 + 实施意图的专有方法论 + 习惯迁移成本
  - 长期：目标完成案例库形成的飞轮

### 1.3 市场机会（摘要）

- TAM：全球个人成长 App ~$150 亿 · SAM：AI 陪伴 × 成长工具交叉 ~$20 亿 · SOM（Year 1）：~50 万付费用户
- 时机：LLM 人格一致性首次商用；用户对「对话不等于成长」有了反思；后疫情自我提升需求持续

## 4. 产品设计原则

LevelUp 的设计风格继承根仓库 CLAUDE.md 中「天鹅挽歌」美学。产品特有的四条原则：

1. **对话先行**：一切功能都从对话触发。用户不需要学习界面，只需要说话。
2. **系统不打扰**：系统只在被需要或被触发时浮现。不做常驻仪表盘。
3. **记忆即关系**：每次对话都是关系的沉淀。没有记忆就没有产品。
4. **可见的进步**：每一步推进都留下可见痕迹，避免「努力了却什么都没看到」的挫败。

## 6. 核心交互模型

### 6.1 对话是第一界面

- 用户进入 App 的唯一默认落点是对话
- 所有关键操作（新目标 / 更新进度 / 查看路线图 / 复盘）都可用自然语言触发
- 所有 UI 控件都有等价的语言表达——控件是语言的快捷方式，不是语言的替代

### 6.2 Sidebar 是系统容器，不是功能入口

- Sidebar 只展示「用户拥有的东西」（目标），不展示「可以做的事」（功能菜单）
- 点击 Sidebar 中的目标 → 把该目标注入对话上下文，AI 接话
- Sidebar 的作用是「让用户在进入对话前有全局感」，而不是「完成操作」

### 6.3 卡片是系统向对话的投射

- 卡片由 AI 根据对话状态主动插入，用户不手动插卡
- 卡片是单向投射：系统 → 对话。用户的反馈仍然用自然语言
- 卡片是可操作的，但默认折叠最小版本

### 6.4 键盘优先

| 快捷键 | 行为 |
|:---|:---|
| ⌘ K | 快速切换目标（把某目标注入对话上下文） |
| ⌘ / | 命令面板（/新目标 · /复盘 · /路线图 · /归档） |
| ⌘ B | 折叠 / 展开 Sidebar |
| ⌘ Enter | 发送 |
| Shift Enter | 换行 |
| Esc | 关闭当前抽屉 / 覆盖层 |

#### 状态清单

| 状态 | 触发 | 呈现 |
|:---|:---|:---|
| 空 | 新用户 | 锁定对话，跳转 Onboarding |
| 首次完成建档 | 建档刚结束 | AI 基于刚建立的目标主动开场 |
| 日内回访 | 距上次 <24h | AI 直接续聊，不重复问候 |
| 间隔回访 | 24h–7d | AI 开场提及上次未决问题 |
| 长期沉默 | >7d | AI 温和重连，不指责，首句询问状态 |
| AI 思考中 | 请求未完成 | 打字指示符 → 3s 后「正在组织语言」 |
| AI 失败 | 请求异常 | 保留用户输入，消息下出现重试按钮 |
| 离线 | 无网络 | 顶部横幅，输入区禁用 |

#### 输入区能力

- 纯文本（MVP）
- 命令前缀 `/`：`/新目标` · `/复盘` · `/路线图` · `/进度` · `/归档`
- `@` 提及目标（Phase 2）
- 消息 24h 内可编辑，编辑后标注已编辑

#### 消息类型

- 用户文本
- AI 文本
- AI 消息中嵌入的结构化卡片（见 F4）
- AI 主动总结（对话结束或 30 分钟无交互后生成一条「本次对话行动项」）

#### 成功指标

- Day 7 留存 > 45% · Day 30 留存 > 25% · 周均对话 ≥ 3 次 · NPS > 40

### F2 · 目标层级树构建

#### 层级结构

```
Level 0   人生使命（Why · 可选）
 └─ Level 1   年度目标（What · 3–5 个）
     └─ Level 2   里程碑（Milestone · 每目标 2–4 个）
         └─ Level 3   本周行动（Action · 每里程碑 2–3 个）
             └─ Level 4   实施意图（当 X 时我就 Y）
```

#### 触发路径

- **首次使用**：强制进入建档向导
- **对话中**：用户说「我想开始一个新目标」→ 进入简化建档
- **Sidebar**：点击「+ 新目标」→ 进入简化建档
- **命令**：`/新目标`

#### 建档向导 · 5 步（目标 3 分钟完成）

**Step 1 · 锚定意义**
- AI：你现在最想推进的一件大事是什么？
- 输入：一句话
- 判断：若过小（任务级）→ 反问「这件事服务于什么更大的目标？」

**Step 2 · 定义完成**
- AI：三个月后，当你觉得「这件事我做到了」时，具体会是什么样？
- 输入：可观察的完成标准
- AI 动作：抽取 1–2 个量化描述候选供用户选择或修改

**Step 3 · 拆里程碑**
- AI：从现在到那个状态之间，有哪些明显的里程碑？
- 形式：AI 先给出 3–4 个候选，用户增删改
- 每个里程碑绑定一个粗略时间（第 X 周）

**Step 4 · 聚焦本周**
- AI：这周你打算先推进哪一个？
- 从 Step 3 结果中选一个作为本周焦点，生成 1–2 个 Action

**Step 5 · 绑定情境（实施意图）**
- AI：你每天大概在什么时间 / 状态下可以投入？
- 生成 1 条实施意图（见 F3），用户确认

**结果**：创建 1 Goal + 2–4 Milestone + 1–2 Action + 1 ImplementationIntention，自动进入对话主界面。

#### 边界

| 情况 | 处理 |
|:---|:---|
| 用户拒绝深入建档 | 只创建 Level 1 + Level 3，Level 2 / 4 延后 |
| 用户输入多个目标 | AI：先聚焦一个，其他的我记下来待会儿 |
| 输入的是任务而非目标 | AI 反问上位目标 |
| 目标超过 1 年 | AI 建议拆为阶段性目标（3–6 个月粒度） |

#### 成功指标

- 首次建档完成率 >60% · 平均层级深度 >3 · 拆解用户 30 日留存差 >20%

### F3 · 实施意图生成

#### 输出模板

```
当 [触发条件]，
我就 [具体动作]，
直到 [结束信号]。
[备选方案 · 可选]
```

#### 生成规则

- **触发条件**：必须是可识别的外部或内部信号（时间 / 地点 / 事件 / 情绪）。越具体越好。
- **具体动作**：动词开头，第一步能在 5 分钟内启动（「打开文档」而非「写作」）
- **结束信号**：可观察（25 分钟 / 写 300 字 / 章节完成），不能是「感觉做完了」
- **备选方案**：第二次漏做时自动生成

#### 存储与生命周期

- 一个 Action 最多绑定 1 条有效实施意图
- 历史版本保留（便于分析什么触发条件对该用户有效）
- 成功 / 失败次数自动累积，用于长期趋势

#### 示例

```
行动：写产品需求文档
↓
当 [周二、周四晚上 9 点，我坐到书桌前]，
我就 [打开需求模板，先填「问题定义」一节]，
直到 [25 分钟番茄钟结束]。
备选：[如果那两天实在累，周六上午 10 点补一次]
```

#### 成功指标

- 有 Action 的用户生成实施意图率 >70%
- 有实施意图 vs 无，行动 7 日执行率差 >30%

### F4 · 对话内嵌卡片

#### 卡片类型（5 种 · MVP）

| # | 类型 | 触发 | 核心信息 |
|:--|:---|:---|:---|
| C1 | 进度更新 | 用户反馈完成 / 推进 | 目标名 + 进度条 + 下一步 |
| C2 | 位置定位 | 用户问「我现在在哪里」 | 目标树中的当前节点高亮 |
| C3 | 目标状态 | 用户提到某目标名 | 该目标当前里程碑 + 最近动作 |
| C4 | 全局摘要 | 对话间隔 >3 天的首条回复 | 所有活跃目标的一行摘要 |
| C5 | 鼓励回顾 | 情绪词识别（累 / 迷茫 / 想放弃） | 过去 30 天完成的行动清单 |

#### 通用结构

```
┌─────────────────────────────┐
│ 🏷 类型                       │ ← 图标 + 标签
│                              │
│ 核心信息（1–3 行 · 大字）     │
│ 次要信息（1–2 行 · 灰字）     │
│                              │
│ ▸ 单一操作                    │ ← 最多一个主操作
└─────────────────────────────┘
```

#### 密度规则

- 单次对话最多 2 张卡片
- 相邻卡片至少间隔 3 条消息
- 同一目标同类卡片 1 小时内不重复

#### 交互

- 默认折叠态（~3 行高度）
- 点击展开完整信息（在对话内展开，不弹窗）
- 主操作直接触发行为，不跳页
- 卡片不可被用户手动添加 / 删除，只能被 AI 决策

#### 成功指标

- 卡片展开率 >40% · 有卡片的对话后「下次打开间隔」显著缩短

### F5 · Sidebar 目标列表

#### 两态

**折叠态（默认，48 px 宽）**
- 只显示目标图标 + 状态色点
- 悬停显示目标名 tooltip

**展开态（280 px 宽）**
- 顶部：「+ 新目标」按钮
- 目标卡片列表：
  - 目标名（1 行）
  - 当前里程碑（灰字 1 行）
  - 细进度条
  - 右上角状态标签
- 底部：排序切换（最近推进 / 创建时间 / 进度）

#### 状态标签

| 标签 | 含义 |
|:---|:---|
| 🟢 推进中 | 7 天内有动作 |
| 🟡 本周未动 | 7 天无动作但未超阈值 |
| 🔴 卡住 | >14 天无动作或连续 3 次漏做 |
| ⭐ 接近完成 | 进度 >80% |
| 📦 已归档 | 仅在筛选「已归档」时出现 |

#### 操作

- 单击：把该目标注入对话上下文，AI 接话「我们聊聊 X」
- 右键 / 长按：编辑 · 归档 · 删除
- 拖动：重排（影响排序 = 自定义时）

#### 边界

- 0 目标：空态文案「让我们从你最重要的一件事开始」+ 主 CTA
- >5 活跃目标：提示「活跃目标过多会稀释推进力，建议聚焦 3 个」（软提醒，不强制）

### F6 · AI 记忆系统

#### 记忆层级

| 层 | 内容 | 维护方式 | 用户可见 |
|:---|:---|:---|:---|
| L1 · 核心档案 | 姓名、城市、职业、价值观、长期关系 | 建档时 + 对话提取 | 是，可编辑 |
| L2 · 目标状态 | 完整目标树 + 进度 + 最近行动 | 实时同步 | 是，通过 Sidebar |
| L3 · 对话摘要 | 每次对话 3–5 条要点 + 情绪标签 + 未决问题 | 对话结束后异步 | 设置页可查看 |
| L4 · 长期趋势 | 执行模式、情绪趋势、反复障碍 | 每 7 天聚合 | 仅 AI 内部使用 |

#### 记忆注入（每次对话前）

按优先级填充上下文，总预算 ~2000 tokens：

1. 核心档案（~300）
2. 活跃目标树摘要（~500）
3. 本次对话前序消息，最近 10 条（~800）
4. 最近 3 次 Session Digest（~400）
5. 根据用户消息语义检索的相关历史片段（~300）

超预算时按顺序截断，1–3 必保。

#### 记忆更新

- **异步更新**：对话结束 30 秒后，LLM 提取结构化摘要，写入 L3
- **周聚合**：每 7 天合并为 L4
- **冲突检测**：用户说「我已经不做 X 了」→ 立即更新 L2 并标注 L3
- **重要度衰减**：L3 记忆每 30 天降权一次，长期趋势不降权

#### 用户可控性

- 设置页「AI 记得什么」面板：
  - L1 所有条目可编辑 / 删除
  - L3 可查看 / 删除（删除时警告「可能影响对话连贯性」）
  - L4 只展示总结，不展示原始数据
- 「忘掉这件事」命令：对话中直接说，触发指定记忆软删除

#### 成功指标

- 第 3 次对话后，用户主动提到「它记住我了」比例 >50%
- 有记忆 vs 无记忆，Day 14 留存差 >25%

## 9. 核心用户旅程

### J1 · 首次使用（~5 分钟）

1. 落地 → 登录（邮箱 / Google）
2. 欢迎页：一句话介绍 + 「开始」（不列功能）
3. AI 开场：「嗨，我是 [AI 名]。我想先知道你现在最想推进的一件大事是什么。」
4. 用户输入初始目标
5. 建档向导 5 步（见 F2）
6. AI 生成目标卡片 + 第一条实施意图，嵌入对话
7. AI：「接下来，我想听你每天的进展。我在这里等着。」

### J2 · 日常回访（<2 分钟）

1. 打开 App → 对话
2. AI 基于时间和上次对话主动开场
3. 用户反馈进展
4. AI 回应 + 可能插入 C1 进度卡片
5. 用户继续 or 关闭

### J3 · 本周复盘（主动触发，~10 分钟）

1. 用户说「帮我复盘这周」或 AI 在周日主动触发
2. AI 展示 C4 全局摘要卡片
3. 依次走过：做了什么 · 没做什么 · 为什么 · 下周调整什么
4. AI 生成下周 Action 清单
5. 更新 Level 3 和实施意图

### J4 · 目标调整

1. 用户：「我想改一下这个目标」
2. AI 确认是哪个目标
3. 区分类型：
   - 进化 → 修改完成标准 / 时间
   - 拆分 → 拆成两个并行目标
   - 合并 → 合入已有目标
   - 放弃 → 归档（询问原因，用于长期趋势）
4. 执行变更 → 同步目标树 + Sidebar

### J5 · 迷失重锚（情绪触发）

1. 用户语言出现：累 / 没意思 / 不知道 / 想放弃
2. AI 不立刻给方案，先共情
3. 插入 C5 鼓励回顾卡片：过去 30 天完成清单
4. AI 问：「你现在是对目标不确定，还是对路径不确定？」
5. 根据回答 → 进入 J4 目标调整 或 路径调整流程

### J6 · 里程碑完成

1. 用户反馈里程碑达成
2. AI 确认 → 更新状态
3. 庆祝时刻：简短过渡动画 + 真诚肯定（不说「太棒了」）
4. AI 引导回顾：「这个里程碑你最意外的发现是什么？」
5. 自然过渡到下一个里程碑

## 11. AI 对话与记忆设计

### 11.1 人格规范（System Prompt 核心条款）

- **身份**：你是用户的成长伙伴。不是助手、不是教练、不是角色扮演。你像一个「认识用户很久、陪他走过几个节点的人」。
- **语气**：温暖但直接。禁止开头语：「当然可以」「好问题」「希望对你有帮助」「让我们一起」。
- **边界**：
  - 不提供医疗 / 法律 / 财务建议
  - 不评判用户的价值观选择
  - 不承诺结果，只承诺陪伴
- **立场**：当用户在自我合理化、逃避、拖延时，要温和但明确地指出。不做和稀泥。
- **记忆呈现**：自然带入对用户的记忆，但不堆砌、不复述「根据我对你的了解…」
- **长度**：默认短回复（<100 字）。深度话题可长，但先判断用户此刻要的是「话」还是「答案」。

### 11.2 意图识别（对话路由）

用户消息到达后先做分类：

| 意图 | 信号 | AI 行为 |
|:---|:---|:---|
| 进展反馈 | 「完成了」「写了」「做到」 | 更新 Action 状态 + 可能 C1 |
| 情绪表达 | 「累」「迷茫」「想放弃」 | 进入共情模式 + 可能 C5 |
| 目标询问 | 「我现在该」「下一步」 | 定位并回应 + 可能 C2 |
| 新目标 | 「我想开始」「新的目标」 | 简化建档流程 |
| 复盘请求 | 「复盘」「总结」「回顾」 | 进入 J3 |
| 目标调整 | 「想改」「不做了」 | 进入 J4 |
| 闲聊 | 其他 | 保持陪伴，轻量回应 |

### 11.3 系统浮现触发表（卡片决策）

| 触发信号 | 卡片 | 优先级 |
|:---|:---|:---|
| 消息含具体完成行为 | C1 进度更新 | 高 |
| 消息含位置 / 下一步类问题 | C2 位置定位 | 高 |
| 消息命中某目标关键词 | C3 目标状态 | 中 |
| 对话间隔 >3 天的首条回复 | C4 全局摘要 | 中 |
| 情绪词命中 | C5 鼓励回顾 | 高 |

触发冲突时：按优先级 → 按新近度 → 按密度规则（见 F4）。

### 11.4 记忆检索策略

见 F6「记忆注入」。

## 12. 空状态与边界

| 场景 | 处理 |
|:---|:---|
| 新用户未完成建档 | 锁定对话，强制引导 |
| 活跃目标数 = 0 | Sidebar 空态：「当你准备好新的旅程时告诉我」 |
| 连续 7 天未对话 | 下次打开，AI 温和重连，不指责 |
| AI 生成失败 | 保留用户输入，消息下出现「让我再想想」+ 重试 |
| 用户删除目标 | 二次确认，30 天内可恢复，超期物理删除 |
| 用户要求删除全部数据 | 设置中提供导出 + 一键删除 |
| 付费到期 | 保留只读访问历史，新建 / 编辑被锁 |
| 同一 Action 连续 3 次漏做 | AI 主动发起微调对话，询问是否调整实施意图 |
| 用户连续 3 天完成行动 | AI 在对话中表达真诚肯定（不触发庆祝动画）|

## 14. 商业模式

### 14.1 模式

免费增值 → 月度订阅 SaaS。定价锚点：真人教练 $500+/月，提供 1/20 价格 + 更高可及性。

### 14.2 定价

| 方案 | 价格 | 范围 | 用户 |
|:---|:---|:---|:---|
| 免费 | $0 | 3 个目标上限、无跨会话记忆、无路线图 | 试用 |
| 成长版 | $19/月 | 无限目标、完整记忆、卡片系统 | P0 主力 |
| 突破版 | $39/月 | 成长版 + 路线图 + 每周 AI 复盘 | 重度 |

⚠️ 待 Van Westendorp 验证。

### 14.3 单位经济（估算）

| 指标 | 值 | 信心 |
|:---|:---|:---|
| CAC | $30–50 | 低 |
| ARPU | $19/月 | 中 |
| LTV（12 月留存 30%） | $85 | 低 |
| LTV / CAC | 1.7–2.8 | 低 |

结论：MVP 阶段留存率是第一优先级，优于增长。
